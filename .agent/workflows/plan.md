---
description: 根據輸入（檔案、GitHub Issue 或純文字描述）規劃繁體中文的實作計劃
---
1. **載入 Skill** (Optional): 
    - 檢查是否設定了以下 Skill:
        - `create-plan`
    - 若有，請讀取，並確保計畫符合該 Skill 定義的架構、設計模式或規範。
2. **分析輸入類型**：
   - **檔案**：如果輸入包含檔案路徑，請使用讀取該檔案內容以理解現有程式碼或文件上下文。
   - **GitHub Issue** (如 `#123`)：如果輸入是 `#` 開頭的編號，請嘗試使用 `gh issue view <number> --json title,body,comments` (如果 `gh`指令可用) 來獲取 issue 的完整規格與討論。如果無法執行 `gh`，請請求使用者提供 issue 內容。
   - **純文字描述**：如果輸入是功能描述或需求說明，直接進行分析。
3. **制定實作計劃**：
   根據獲取的資訊，撰寫一份詳細的**繁體中文**實作計劃。請使用以下結構：
   # 實作計劃：[標題]
   ## 1. 目標 (Goal)
   簡述要解決的問題或要達成的功能目標。
   ## 2. 背景與分析 (Context & Analysis)
   - 分析現有的程式碼結構或問題成因。
   - 如果是 Issue，摘要 Issue 的核心需求。
   - 列出任何潛在的技術挑戰或相依性。
   ## 3. 預計變更 (Proposed Changes)
   列出具體的執行步驟，例如：
   - **[MODIFY]** `path/to/file`: 說明修改邏輯
   - **[NEW]** `path/to/new/file`: 說明新檔案用途
   - **[REMOVE]** `path/to/file`: (若有)
   ## 4. 驗證計畫 (Verification Plan)
   - 描述如何驗證變更是否成功 (例如：新增單元測試、手動測試流程)。
4. **輸出確認**：
   將此計劃呈現給使用者，並詢問是否需要調整，或直接開始執行，或是否要輸出到某個檔案中。
