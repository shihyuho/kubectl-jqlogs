description: 根據需求，擬定繁體中文的實作計劃
---

1. **識別需求來源：**

判斷使用者提供的需求是純文字還是檔案路徑，如果是檔案路徑，則讀取其內容。如果是純文字，則直接使用提示中的文字。

2. **識別輸出方式：**

判斷使用者提供的需求內容是否包含輸出規則，例如輸出到某個指定檔案，若沒有，則結果以 Markdown 格式提供給使用者複製

3. **分析與拆解：**

深入分析需求內容，以識別核心功能、使用者故事、技術限制和成功標準。將大型任務分解為一系列更小、可管理的子任務。

4. **建立結構化計劃：**

考慮任務之間的依賴關係，將子任務組織成一個邏輯清晰的執行序列。對於複雜的項目，可以將相關的子任務分組到較大的史詩 (Epic) 或里程碑 (Milestone) 之下。

5. **產出計劃：**

- 使用繁體中文來表達最終的實作計劃。計劃中的每一個項目都應該是一個清晰、簡潔的待辦事項，方便追蹤與執行。
- 以 「識別輸出方式」 的結果將實作計劃呈現給使用者
